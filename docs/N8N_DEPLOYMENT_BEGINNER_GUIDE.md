# n8n導入ガイド（非推奨化）

このドキュメントは互換運用のために残していますが、現行アーキテクチャでは n8n を使いません。

## 現行方針

- 外部アクションは Bot コード内で直接実行します。
- 通常は `/ask` から実行します（内部で必要なactionを自律的に判断）。
- 手動デバッグ時のみ `/debug_action` を使用します。
- 認証状態の確認は `/auth_status` を使用します。
- 認証が必要なアクションは `auth_required` と `auth_url` を返すため、Discordに表示されたURLから設定を完了してください。

## 参照先

- 運用手順: `README.md` の「外部アクション実行（コード内）」
- 仕様: `docs/DESIGN.md` の 3.2 節（内部アクション実行ツール）

## 補足

既存環境で n8n コンテナを動かしていても、現行コードは n8n を呼び出しません。段階的に停止・撤去して問題ありません。
